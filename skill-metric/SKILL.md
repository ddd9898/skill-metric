# Skill Metric — Skill Quality Evaluation

Runs **static quality scoring** on agent skill directories: format, completeness, and writing scores (24 pts max), with optional JSON, CSV, and radar chart output.

## When to Use This Skill

Use this skill when the user wants to:
- Score one or more skills (e.g. under `skills/`) and see format / completeness / writing scores
- Audit skill quality or get a detailed pass/fail report per check
- Export all scores to a CSV for statistics or dashboards
- Generate a radar chart for a single skill (`--figure`)
- Get machine-readable output (JSON) for downstream tooling

## How to Run

The evaluation script is in `scripts/skill_quality_eval.py`. Paths are resolved relative to the **current working directory**.

```bash
# Score one skill (full report)
python skills/skill-metric/scripts/skill_quality_eval.py skills/uniprot-database

# Score all skills under skills/ and write CSV
python skills/skill-metric/scripts/skill_quality_eval.py skills/*/ --csv scores.csv

# JSON output
python skills/skill-metric/scripts/skill_quality_eval.py skills/uniprot-database -j

# Radar chart (single skill only, requires matplotlib)
python skills/skill-metric/scripts/skill_quality_eval.py skills/uniprot-database --figure
```

### Options

| Option | Description |
|--------|-------------|
| `-q`, `--quiet` | Print only total scores |
| `-j`, `--json` | Output JSON |
| `--csv` [file] | Output CSV; no path → stdout, with path → save to file |
| `--figure` [file] | Radar chart PNG (single skill only) |

### Calling from Python (subprocess)

**CRITICAL**: Shell globs like `skills/*/` are NOT expanded when using `subprocess` with a list. Use `glob.glob()` to expand paths first:

```python
import glob, subprocess
skill_dirs = sorted(glob.glob('/path/to/skills/*/'))
subprocess.run(
    ['python', 'skills/skill-metric/scripts/skill_quality_eval.py']
    + skill_dirs + ['--csv', '/path/to/output/scores.csv'],
    check=True
)
```

See `references/scoring_criteria.md` for more Python examples and full output format details.

## Scoring Summary (24 pts max)

| Dimension | What is checked | Max |
|-----------|----------------|-----|
| **2.1.1 Format** | SKILL.md exists, naming, frontmatter, description validity | 8 |
| **2.1.2 Completeness** | license, compatibility, metadata, scripts·references·assets dirs, examples, error handling | 8 |
| **2.1.3 Writing** | Task boundary, trigger, progressive disclosure (≤5k chars), English, ref consistency, version | 8 |

Full per-check details: see `references/scoring_criteria.md`.

## Error Handling / Troubleshooting

- **Path not found**: If a path does not exist or is not a skill directory/SKILL.md, the script prints `Error: Path not found or not SKILL.md / dir: <path>` to stderr and records an error entry. Other paths in the same run continue to be evaluated.
- **Shell glob not expanded** (Python subprocess): Passing `*/` as a literal string causes "path not found". Always use `glob.glob()` to expand before calling subprocess. See "Calling from Python" above.
- **`--figure` fails**: Requires `matplotlib` (`pip install matplotlib`). Only supported for a single skill; passing multiple skills with `--figure` is ignored.
- **`skills/` treated as one skill**: Passing the parent directory (e.g. `skills/`) is treated as one skill named "skills". Pass `skills/*/` or individual subdirectories instead.
